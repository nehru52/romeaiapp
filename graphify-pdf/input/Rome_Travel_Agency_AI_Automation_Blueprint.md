awareness to booking. Views don't pay commissions—bookings do.

$5,000+/month on creative teams, while operating at a fraction of the cost. The system doesn't just
post—it learns, adapts, and optimizes based on real performance data, creating a compounding

traditional scheduling tools that simply post pre-written content, Eliza OS is agent-native: it can scrape
trends, generate responses, manage multiple brand personas, and operate 24/7 without human
intervention.

 Multi-character management: Run different personas for different platforms or client segments
 Autonomous engagement: Reply to comments, answer DMs, and participate in relevant
conversations
 Memory and learning: Remember past interactions, top-performing content, and audience
preferences
 Plugin ecosystem: Connect to image APIs, video APIs, scheduling tools, and analytics platforms
 Cost: Completely free and open-source. Requires a $5/month VPS or can run locally.
Setup:
1. Clone repository: git clone https://github.com/ai16z/eliza
2. Configure character file with your Rome travel agency brand voice
3. Set DeepSeek V4 as the LLM backend (OpenAI-compatible API)
4. Add plugins for image generation, video generation, and scheduling
5. Deploy on VPS or local machine

8. TREND DETECTION: AUTOMATED INTELLIGENCE
DeepSeek V4 cannot browse the internet. It is a reasoning engine, not a browser. For real-time trend
detection, you need a data collection layer that feeds live social media data into DeepSeek for analysis.
Option 1: Apify (Recommended)
Cloud platform with pre-built scrapers. TikTok Scraper Actor runs on schedule, extracts videos, captions,
hashtags, music, and metrics. Outputs JSON for DeepSeek analysis. Cost: $0.29 per 1,000 results.
Scheduled daily at 6:00 AM.
Option 2: SocialCrawl API
Unified API for 42 platforms. One call returns 50 trending posts with engagement rates, velocity scores,
and content categorization. 100 free credits to start. Cost: ~$0.01-0.05 per request after free tier.
Option 3: Open Source (Free)
Firecrawl + TrendFinder (GitHub). Self-hosted TypeScript stack. Monitors websites and Twitter. Requires
4-8 hours setup and ongoing maintenance when platforms change their HTML structure.
The Workflow:
1. 6:00 AM: Apify/SocialCrawl scrapes trending Rome content automatically
2. 6:05 AM: DeepSeek V4-Pro analyzes the JSON data, extracts viral patterns
3. 6:10 AM: Slack alert: 'New trend brief ready for review'
4. 7:00 AM: Eliza OS generates content based on the brief
5. 9:00 AM: Auto-posts to all platforms at optimal times

11. THE BOOKING CONVERSION FUNNEL
Views don't pay commissions—bookings do. The funnel is designed to capture high-intent travelers at the
moment of peak interest and nurture them through to a consultation call.
Stage 1: AWARENESS (Viral Content)
Reels and carousels generate reach and saves. The goal is not to sell yet—it is to become the Rome
travel expert in the viewer's mind. Content must be 60% inspirational, 30% educational, 10% promotional.
Stage 2: INTEREST (Profile Binge)
When a viral post catches attention, the viewer visits your profile. They must find 5-10 pieces of consistent,
high-quality content that builds trust. This is why daily posting matters—an empty profile kills conversion.
Stage 3: CAPTURE (Lead Magnet)
Link in bio leads to a landing page offering a free, high-value resource. Options: '7-Day Rome Itinerary
PDF,' 'Italy Budget Calculator,' 'Rome Hidden Gems Map,' or 'Best Time to Visit Italy Guide.' The resource
must be genuinely useful—travelers can smell a worthless lead magnet.
Stage 4: NURTURE (Email Sequence)
Day 1: Deliver the lead magnet + introduce the agency with a personal story. Day 3: Client success story +
social proof (photos, testimonials, specific results). Day 5: Educational content—'5 Common Rome
Mistakes That Cost Tourists Hundreds.' Day 7: Limited-time offer or early booking discount with urgency.
Day 10: Direct booking CTA with Calendly link for a free 15-minute consultation.
Stage 5: CONVERSION (Consultation → Booking)
The consultation is not a sales pitch—it is a discovery call to understand the traveler's needs. The goal is
to design a custom itinerary that feels personalized. Deposit secures the booking. UGC collected during
the trip feeds back into the content cycle.

16. TROUBLESHOOTING & MAINTENANCE
Common Issues:
Issue: Apify scraper stops working
Cause: TikTok updated their anti-bot protection or HTML structure
Fix: Apify Actors are maintained by the community—updates usually arrive within 24-48 hours. Check
the Actor's GitHub page for updates. Alternative: temporarily switch to SocialCrawl API.
Issue: DeepSeek outputs generic content ideas
Cause: Not enough specific data fed into the prompt
Fix: Include actual scraped post data (captions, hashtags, view counts) in the prompt. The more
specific the input, the more specific the output.
Issue: AI-generated images look too artificial
Cause: Prompt too generic or wrong model selected
Fix: Add specific camera/lens references (Sony A7R V, 35mm f/1.4) to FLUX.2 prompts. Use
photorealistic style keywords. For UGC-style content, add 'authentic phone camera aesthetic, slight
motion blur.'
Issue: Low engagement despite high posting frequency
Cause: Content not aligned with current trends or algorithm preferences
Fix: Review trend scraper data—is the content actually reflecting what's viral? Check first 3 seconds
of every Reel. If the hook doesn't stop the scroll, nothing else matters.
Issue: DMs and comments not converting to bookings
Cause: Response too slow or too salesy
Fix: Eliza OS should reply within 60 minutes (ideally 15). DeepSeek V4-Flash should be trained to
provide value first, ask qualifying questions second, and suggest a consultation third—not pitch
immediately.
