# Project Subfolder Completeness Checklist

**Template Version:** 2.1  
**Applicable Standard:** Folder Naming Convention v2.3  
**Last Updated:** 2025-01-10

---

## Section A: Folder Name Verification

- [ ] **Project code prefix confirmed** — Folder name starts with the official project code (e.g., `05TK25002D`)
- [ ] **Naming pattern correct** — Folder follows `{project_code}-{project_name}` format with a hyphen separator
- [ ] **Year folder exists** — The appropriate `{year} Projects` parent folder is present under `D:\My Projects\`
- [ ] **No duplicate folders** — No other folder exists for the same project under any year folder
- [ ] **Isolation policy acknowledged** — Project folder is self-contained; no cross-project references or shortcuts

---

## Section B: Mandatory Subfolder Checklist

Verify that all 10 mandatory subfolders have been created inside the project folder:

- [ ] `01_Project_Management` — Project plans, schedules, budgets, resource allocation
- [ ] `02_Design_Documents` — Design basis, specifications, technical standards
- [ ] `03_Calculations` — Process calculations, equipment sizing, structural analysis
- [ ] `04_Drawings` — PFDs, P&IDs, plot plans, GA drawings, isometrics
- [ ] `05_Procurement` — Vendor quotes, purchase orders, material requisitions
- [ ] `06_Correspondence` — Client communications, internal memos, transmittals
- [ ] `07_Meeting_Minutes` — Kick-off meetings, design reviews, progress meetings
- [ ] `08_Quality_Control` — QA/QC procedures, inspection reports, audit records
- [ ] `09_Deliverables` — Final compiled deliverable packages for client submission
- [ ] `10_Archive` — Superseded documents, historical records, reference materials

---

## Section C: Initial Setup Verification

- [ ] **README.md placed in project root** — Contains project overview, team contacts, and folder guide
- [ ] **Folder permissions set** — Only authorized project team members have write access
- [ ] **Backup verification** — Confirmed that the new folder is included in the weekly backup scope
- [ ] **Project code cross-referenced** — Verified against the official Company Projects Registry (JSON)

---

## Section D: Ongoing Documentation Requirements

### Milestone Tracking
- Track each project milestone in `01_Project_Management`. Maintain a milestone log that records:
  - Milestone description
  - Planned date vs. actual date
  - Responsible person
  - Status (Pending / In Progress / Completed / Delayed)

### Deliverable Compilation
- Record deliverable compilation requirements in `09_Deliverables`. For each deliverable package:
  - Document the deliverable list and revision status
  - Maintain a transmittal register
  - Archive superseded versions in `10_Archive` with clear version labels

### Progress Summaries
- Summarize progress periodically (recommended: bi-weekly) in `01_Project_Management`:
  - Overall project status
  - Key accomplishments since last summary
  - Upcoming activities and deadlines
  - Issues and risks requiring attention
  - Resource utilization notes
