# Linux BSP

Linux BSP artifacts must consume `sw/platform/e1_platform_contract.json` or generated headers from it for e1 MMIO base addresses, IRQ names, and register offsets.

`make linux-bsp-check` rejects a documentation-only BSP. The first Linux-facing artifacts are:

```text
sw/linux/drivers/e1/Kconfig
sw/linux/drivers/e1/Makefile
sw/linux/scripts/import-linux-bsp.sh
sw/linux/configs/eliza_e1.fragment
sw/linux/dts/eliza-e1.dts
sw/linux/dts/Makefile
sw/platform/generated/e1-platform.dtsi
sw/linux/Documentation/devicetree/bindings/eliza/eliza,e1-*.yaml
sw/linux/drivers/e1/e1_platform_contract.h
sw/linux/drivers/e1/e1-npu-uapi.h
sw/linux/drivers/e1/e1-npu.c
sw/linux/drivers/e1/e1-dma.c
sw/linux/tests/e1-mmio-smoke.c
sw/linux/tests/e1-npu-smoke.c
NPU char driver
DMA test driver
```

The WiFi/Bluetooth BSP slice is a disabled DTS scaffold for an external Murata Type
1DX / CYW4343W-class module shape. It names the intended SDIO `brcmfmac` WiFi
device, UART `hci_uart_bcm` Bluetooth transport, and power sequencing hook, but
does not claim an implemented e1-chip SDIO host, UART, GPIO, pinctrl, or RF
block.

## Repo-local scaffold check

Command:

```sh
make linux-bsp-check
python3 sw/check_bsp_scaffolds.py linux
```

Expected output:

```text
linux: scaffold audit
  local command: make linux-bsp-check
  expected output: linux BSP check passed.
  dependency blocker: external Linux kernel checkout plus integration of drivers/misc/eliza-e1
  status: clear
linux BSP check failed:
  - linux BSP BLOCKED: missing evidence for external Linux kernel build, DTB validation, and runtime driver smoke transcript: docs/evidence/linux/eliza_e1_kernel_build.log, docs/evidence/linux/eliza_e1_dtb_check.log, docs/evidence/linux/e1-mmio-smoke.log
```

Dependency blocker: a real Linux boot/image check requires an external kernel
checkout with `sw/linux/drivers/e1` imported under
`drivers/misc/eliza-e1`, `sw/linux/dts/eliza-e1.dts` copied into
`arch/riscv/boot/dts/eliza`, the Eliza DT bindings copied under
`Documentation/devicetree/bindings/eliza`, and board-owned
Kconfig/Makefile fragments that select `CONFIG_ELIZA_E1_NPU` and
`CONFIG_ELIZA_E1_DMA`.

Evidence intake is defined by
`docs/evidence/software-bsp-evidence-manifest.json` and validated by
`make software-bsp-evidence-check`. A transcript must be generated by the
capture path or match its `eliza-evidence` markers, command markers, and
pass markers. Templates, substitute-only logs, failed transcripts, and too-small files
are rejected.

Environment readiness can be checked without creating evidence logs:

```sh
python3 scripts/check_software_bsp.py external-preflight linux \
  --linux /path/to/linux \
  --target-host root@TARGET \
  --write-report
```

## External Linux import

The checked-in driver directory is ready to copy into an external Linux tree:

```sh
sw/linux/scripts/import-linux-bsp.sh /path/to/linux
mkdir -p /path/to/linux/drivers/misc/eliza-e1
rsync -a sw/linux/drivers/e1/ /path/to/linux/drivers/misc/eliza-e1/
cp sw/linux/dts/eliza-e1.dts sw/linux/dts/Makefile \
  /path/to/linux/arch/riscv/boot/dts/eliza/
cp sw/platform/generated/e1-platform.dtsi \
  /path/to/linux/arch/riscv/boot/dts/eliza/
cp sw/linux/Documentation/devicetree/bindings/eliza/*.yaml \
  /path/to/linux/Documentation/devicetree/bindings/eliza/
cp sw/platform/generated/e1_platform_contract.h /path/to/linux/drivers/misc/eliza-e1/e1_platform_contract.h
cp sw/linux/configs/eliza_e1.fragment /path/to/linux/kernel/configs/eliza_e1.config
```

Then add these external-tree fragments:

```make
# drivers/misc/Makefile
obj-$(CONFIG_ELIZA_E1_BSP) += eliza-e1/
```

```text
# drivers/misc/Kconfig
source "drivers/misc/eliza-e1/Kconfig"
```

```make
# arch/riscv/boot/dts/Makefile
subdir-y += eliza
```

The repo-local helper validates that the destination looks like a kernel tree
and prints the import commands. It does not patch or build the external kernel.

Expected helper output starts with:

```text
Import commands:
  mkdir -p /path/to/linux/drivers/misc/eliza-e1 ...
```

## External evidence capture

From this repository, with `/path/to/linux` already imported and configured:

```sh
sw/linux/scripts/capture-linux-bsp-evidence.sh /path/to/linux kernel-build
sw/linux/scripts/capture-linux-bsp-evidence.sh /path/to/linux dtb-check
E1_SMOKE_CMD='ssh root@TARGET /usr/bin/e1-npu-smoke --device /dev/e1-npu' \
  sw/linux/scripts/capture-linux-bsp-evidence.sh /path/to/linux smoke
make software-bsp-evidence-check
```

Set `CROSS_COMPILE=riscv64-linux-gnu-` and `JOBS=<n>` if the external kernel
tree needs a specific toolchain or parallelism. `dtb-check` intentionally uses
`dtbs_check`; missing schemas or DTS integration remain blockers, not local
passes.

## Linux boot artifact gate

`make linux-boot-artifacts-check` is the fail-closed gate for the first basic
kernel/rootfs/OpenSBI boot path. It does not build anything by itself and it
does not promote `.BLOCKED` notes into evidence. It checks host preflight,
required external tree environment variables, the Chipyard Linux payload
locator, and the required kernel, DTB, OpenSBI, rootfs/initramfs, boot log, and
smoke transcripts named in
`docs/evidence/linux/eliza-linux-boot-artifacts.json`.

Set these paths before capture:

```sh
export ELIZA_LINUX_TREE=/path/to/linux
export ELIZA_BUILDROOT_TREE=/path/to/buildroot
export ELIZA_OPENSBI_TREE=/path/to/opensbi
```

Exact command plan for a Linux host or CI worker:

```sh
sw/linux/scripts/import-linux-bsp.sh "$ELIZA_LINUX_TREE"
sw/buildroot/scripts/import-buildroot-external.sh "$ELIZA_BUILDROOT_TREE"

sw/buildroot/scripts/capture-buildroot-evidence.sh "$ELIZA_BUILDROOT_TREE" defconfig
make -C "$ELIZA_BUILDROOT_TREE" BR2_EXTERNAL="$PWD/sw/buildroot"
sw/buildroot/scripts/capture-buildroot-evidence.sh "$ELIZA_BUILDROOT_TREE" image-manifest

cd "$ELIZA_LINUX_TREE"
make ARCH=riscv eliza_e1.config olddefconfig
cd -
sw/linux/scripts/capture-linux-bsp-evidence.sh "$ELIZA_LINUX_TREE" kernel-build
sw/linux/scripts/capture-linux-bsp-evidence.sh "$ELIZA_LINUX_TREE" dtb-check

python3 scripts/locate_chipyard_linux_payload.py --require
CHIPYARD_LINUX_BINARY=<selected_payload> scripts/run_chipyard_eliza_linux_smoke.sh
# On PASS, the wrapper writes docs/evidence/linux/eliza_e1_serial_boot.log.

ELIZA_OPENSBI_HANDOFF_CMD='<exact boot command>' \
  docs/sw/opensbi/capture-opensbi-evidence.sh "$ELIZA_OPENSBI_TREE" handoff

E1_SMOKE_CMD='<target command for /usr/bin/e1-mmio-smoke>' \
  sw/linux/scripts/capture-linux-bsp-evidence.sh "$ELIZA_LINUX_TREE" smoke

make linux-boot-artifacts-check
```

If the generated AP transcript also boots the elizaOS image far enough to
prove agent liveness, convert the real transcript into OS-side manifest
evidence from `packages/os/linux/elizaos`:

```sh
scripts/capture-chip-boot-evidence.py \
  --boot-transcript ../../../chip/docs/evidence/linux/eliza_e1_serial_boot.log \
  --agent-transcript /path/to/generated-ap-agent-health.log
```

Current expected local status is `BLOCKED` until the external kernel checkout,
Buildroot images, OpenSBI handoff transcript, selected Chipyard payload, serial
boot transcript, and target smoke log exist with PASS evidence markers.

## /dev/e1-npu ABI

The `eliza,e1-npu` driver registers `/dev/e1-npu`. `read(2)` returns
the current scalar `RESULT` register as text for legacy MMIO smoke scripts.
New Linux smoke code uses the ioctl UAPI in
`sw/linux/drivers/e1/e1-npu-uapi.h`:

```c
E1_NPU_IOC_RUN_CMD       /* scalar opcode, including DOT4_S8 */
E1_NPU_IOC_RUN_GEMM_S8   /* bounded INT8 GEMM tile: 1 <= M,N <= 3, 1 <= K <= 7 */
E1_NPU_IOC_GET_COUNTERS  /* control, descriptor, and perf registers */
```

`E1_NPU_IOC_RUN_CMD` writes `OP_A`, `OP_B`, `ACC`, and `OPCODE`, starts the
block, waits for `DONE`, and returns `RESULT` plus `CTRL_STATUS`.
`E1_NPU_IOC_RUN_GEMM_S8` copies A and B into the 64-byte MMIO scratchpad,
programs the GEMM config/base/stride registers, starts opcode `GEMM_S8`, and
returns C as signed int32 values. Oversized dimensions fail with `-EINVAL`
before hardware launch.

## Target-side NPU smoke

`sw/linux/tests/e1-npu-smoke.c` is source for an external target rootfs.
Build it with the imported UAPI header, install it on the target, then run:

```sh
/usr/bin/e1-npu-smoke --device /dev/e1-npu
```

The smoke runs one `DOT4_S8` scalar command and one fixed 2x2x3 signed INT8
GEMM workload. It prints `eliza-evidence` markers and a PASS line when the
driver returns the expected results. This is driver ioctl smoke only; it is not
NNAPI/TFLite acceleration, descriptor DMA proof, or hardware benchmark
evidence.
