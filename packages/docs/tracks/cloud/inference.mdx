Cloud-routed inference is what an Eliza agent uses when it calls elizaOS Cloud
instead of a provider directly. Cloud acts as a gateway: it accepts
OpenAI-compatible chat requests, resolves the requested model against the live
catalog, enforces rate limits and credits, and returns the response.

## Why route through Cloud

- One API key for all providers.
- Centralized rate limiting, budget caps, and observability.
- Automatic fallback when a provider errors or is over quota.
- Per-app inference markup for creator monetization.
- Cheaper aggregate pricing when Cloud has bulk-rate contracts.

## Model routing

Use `GET /api/v1/models` to inspect the live catalog. Chat requests go through
`POST /api/v1/chat/completions`, Anthropic-compatible clients can use
`POST /api/v1/messages`, and media routes use their dedicated `/api/v1/*`
generation endpoints.

## Fallback

If the primary provider returns a transient error or is rate-limited, Cloud retries against the next configured provider for that model type. The agent sees one logical call.

## Rate limits

Each app has per-minute, per-hour, and per-day caps configurable in the dashboard. Caps hit return `429` to the agent; the agent's local retry logic decides whether to back off or escalate.
