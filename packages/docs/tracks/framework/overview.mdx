The **Framework** track is for developers who want the elizaOS runtime as a library, not as an app. You import `@elizaos/core`, construct an `AgentRuntime` directly, and embed it in whatever surface you already own — a CLI, an HTTP server, a Discord bot, a background worker.

## What you build

A TypeScript application that links against `@elizaos/core`, loads a `Character` and the plugins you need, and drives the runtime programmatically. Nothing else from the elizaOS distribution is assumed — no dashboard, no Electrobun shell, no Cloud.

## Minimal example

```ts
import { AgentRuntime } from '@elizaos/core';
import sql from '@elizaos/plugin-sql';
import anthropic from '@elizaos/plugin-anthropic';

const runtime = new AgentRuntime({
  character: {
    name: 'Eliza',
    bio: 'A helpful agent.',
    plugins: [],
  },
  plugins: [sql, anthropic],
});

await runtime.initialize();
// Send a message:
await runtime.messageManager.handleMessage({ /* ... */ });
```
