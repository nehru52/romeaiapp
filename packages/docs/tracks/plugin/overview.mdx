A **Plugin** bundles a slice of agent capability — actions, providers, evaluators, services, model adapters, schemas, or HTTP routes — into a single npm package that any Eliza agent can load by adding its name to the character's `plugins` array.

## What you build

A package under `@elizaos/plugin-<name>` (or your own scope) that exports a `Plugin` object. When loaded, it registers everything it provides with the runtime.

## Plugin surface

```ts
import type { Plugin } from '@elizaos/core';

export const plugin: Plugin = {
  name: 'plugin-example',
  description: 'What it does in one sentence.',
  actions: [/* ... */],
  providers: [/* ... */],
  evaluators: [/* ... */],
  services: [/* ... */],
  models: { /* ... */ },
  routes: [/* ... */],
  schemas: [/* ... */],
  init: async (runtime) => { /* boot-time setup */ },
};
```

Every field is optional. A plugin can be as small as one action.
