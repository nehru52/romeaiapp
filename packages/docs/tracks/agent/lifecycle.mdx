## Boot

1. `AgentRuntime` is constructed with a `Character` and an array of plugins.
2. The runtime resolves plugins from npm (or workspace), loading each module.
3. Each plugin's `init?(runtime)` is invoked. Plugins register their actions, providers, evaluators, services, models, and routes.
4. Services start. Database adapters connect. Embeddings backfill if needed.
5. `runtime.initialize()` resolves. The agent is live.

## Message flow

For every incoming message:

1. **Connection ensure** — entity, room, and participants are upserted.
2. **Memory write** — the message is persisted with embeddings.
3. **Providers** — registered providers (`messages`, `facts`, `goals`, `enabled_skills`, etc.) produce context strings ordered by `position`.
4. **Action selection** — the runtime asks the model which action(s) to run.
5. **Action handler** — selected actions execute. They can call models, hit APIs, or stream callbacks back to the user.
6. **Evaluators** — post-response evaluators run (fact extraction, reflection, etc.).

## Shutdown

`runtime.stop()` flushes pending writes, releases database connections, stops services, and unregisters connectors. Always await it on process exit.
